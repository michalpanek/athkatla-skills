# Severity Guidelines

- **CRITICAL**: Security issues, data loss risks, broken business logic, `any` type, silent error swallowing, missing auth checks, raw `Error` instead of `ActionError` in actions, missing transaction on multi-entity mutations, N+1 queries in loops, `console.log` in production code, missing input validation on server actions, hardcoded secrets or credentials, unsafe type assertions casting `undefined` to a model type, division by zero without guard, `let` with mutation of shared state, unsanitized user input in RegExp (ReDoS), missing env var validation at startup, `key={Math.random()}` on React lists, deleting parent before dependent entities
- **HIGH**: Wrong package placement, misleading function names hiding side effects (`prepareCache` that saves), missing cache revalidation after mutations, missing `executeQuery` wrapper in repositories, `as` type assertions, non-null assertions, `.then()/.catch()` chains, default exports, missing error context in `ActionError`, direct DB queries without `unstableCache`, `function` declarations instead of arrow functions, missing Zod validation at system boundaries, missing return types on exported functions, index as React list key, using `let` when `const` works, `.push()` mutation instead of spread/concat, `Array.filter` returning non-boolean (truthy arrays), missing cache key parameters in react-query
- **MEDIUM**: Generic variable names (`process`, `prepare`, `trigger`), missing early returns (if/else chains), magic numbers in business logic, inconsistent destructuring patterns, redundant comments that duplicate code, `string[]` instead of `Array<string>`, missing `satisfies` where beneficial, test methods over 100 lines, inline lambda functions in JSX, missing loading state distinction (`isExecuting` vs `isPending`), post-fetch pagination instead of database-level, nullable types not reflected in type signature, inconsistent naming conventions within same PR, `==` instead of `===`, reimplementing library functions, mixed Arrange-Act-Assert in tests
- **LOW**: Inconsistent object constant casing, test resource naming, sub-optimal but working code, missing log entries for non-critical operations, cosmetic formatting, `ReadonlyArray` not used where applicable, minor naming inconsistencies, `React.FC` instead of `FC`, missing shorthand property syntax, unnecessary fragment wrappers
