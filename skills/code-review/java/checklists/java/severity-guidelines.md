# Severity Guidelines

- **CRITICAL**: Security issues, data loss risks, broken business logic, returning null that causes NPE, iterator `.next()` without checking, N+1 queries in loops, `Optional.get()` without check, silent message drops (missing Queue `@Bean`), missing `@Modifying` on delete queries, MDC context leak in `@Scheduled` methods (no servlet filter safety net), missing exception cause in wrapping
- **HIGH**: Wrong package placement, misleading method names hiding side effects, duplicate error codes, missing validation on collections, `@Deprecated` misuse, missing `@Transactional` on multi-write operations, field injection instead of constructor injection, missing guard clauses causing unnecessary processing, domain service calling repository directly (bypassing persistence service), missing `@JsonIgnoreProperties` on external API records
- **MEDIUM**: Generic variable names, missing blank lines in dense methods, magic numbers in tests, deeply nested getter chains not extracted, test methods over 100 lines, inconsistent mapper usage, `@Column` on fields matching default naming, missing MDC `removeXFromLogs()` for within-request clarity, collection fields not initialized in entities
- **LOW**: Inconsistent MDC method naming, test resource file naming, sub-optimal but working code, missing log entries for non-critical operations, cosmetic formatting issues, `LinkedList` vs `ArrayList` choice
